When the user says the model code changed, create a plan to update the corresponding thesis sections to match — do not just describe what changed, identify every place in the LaTeX that needs editing.

**Why:** The thesis must stay in sync with the implementation. Divergence between code and thesis text causes inconsistency that reviewers will catch.

**How to apply:** Always read both `implementation/phabert_cnn_model.py` and the relevant chapter `.tex` file before writing any plan touching architecture sections.
